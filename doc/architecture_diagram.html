vision_grasp Runtime Architecture

Single-package ROS 2 Jazzy visual grasping prototype for Piper. The current runtime is simulation-first: camera generation, color-based detection, pixel-to-base transform, IK-driven trajectory planning, and optional serial actuation. Styled as a standalone HTML/SVG diagram in the spirit of Cocoon-AI's architecture-diagram-generator output.

Runtime Summary

  • Single ROS 2 package with five executable nodes.
  • Simulation-first startup path through pipeline.launch.py and demo.launch.py.
  • Shared parameter file now reaches camera, vision, tf, trajectory, and driver nodes.

Engineering State

  • README, LICENSE, CHANGELOG, dependency declarations, and launch wiring are in place.
  • Package-level tests now cover metadata, IK helpers, TF math, and pipeline smoke launch.
  • The internal detection and grasp contracts still reuse PoseStamped.

Next Safe Refactor

  • Only after that, introduce explicit interface messages for detection and grasp targets.
  • Keep the existing URDF and gripper sign convention unchanged.
